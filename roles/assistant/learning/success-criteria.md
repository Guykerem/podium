# Personal Assistant — Success Criteria

How to evaluate whether each skill is performing well. The role is working when the user feels *lighter*, not more managed.

## check-in

| Rating | Criteria |
|--------|----------|
| **Good** | The user responds positively (or just acknowledges) — the check-in surfaced something they were glad to hear. Interrupt rate is low and acceptance rate is high. |
| **Okay** | User acknowledges but doesn't act. The signal was real but timing or framing was off. |
| **Poor** | User ignores, dismisses, or explicitly asks the agent to stop. The interrupt did not earn the interruption. |

## manage-tasks

| Rating | Criteria |
|--------|----------|
| **Good** | Tasks are captured near-instantly from any channel, overdue items are surfaced before the user notices them, and the user's task system (Todoist/Linear/Notion) matches reality. Weekly review is completed with minimal friction. |
| **Okay** | Tasks are captured but occasionally duplicated, or overdue items slip for a day. System is mostly trusted. |
| **Poor** | User has stopped trusting the task system — they keep a shadow list in their head. Overdue items regularly slip. |

## manage-calendar

| Rating | Criteria |
|--------|----------|
| **Good** | Zero conflicts make it onto the calendar. Deep work blocks survive the week. Scheduling suggestions match the user's energy map and preferences. |
| **Okay** | Occasional double-bookings caught in brief rather than prevented. Suggestions are reasonable but not personalized. |
| **Poor** | User has lost trust — they manually re-check every suggestion. Conflicts reach the inbox of other people. |

## manage-email

| Rating | Criteria |
|--------|----------|
| **Good** | Q1 emails surface same-day. Drafted replies need minimal editing. Newsletter and Q4 noise stays out of the brief. False positives on urgency are rare. |
| **Okay** | Triage is helpful but the user still does a full inbox pass. Drafts need significant rewriting. |
| **Poor** | User doesn't trust the triage — they re-read the inbox from scratch. Drafts miss tone and get discarded. |

## daily-brief

| Rating | Criteria |
|--------|----------|
| **Good** | Brief is under a 2-minute read. Every item earned its place. User acts on at least one thing before closing. Nothing important that day was missing. |
| **Okay** | Brief is useful but long. Some items feel like padding. User skims rather than reads. |
| **Poor** | User has stopped opening the brief, or reports it reminded them of things they'd already done. |

## relationship-coach

| Rating | Criteria |
|--------|----------|
| **Good** | Dormant-relationship nudges feel timely and welcome. Suggested openers land well enough to use with light edits. Cadence targets hold. |
| **Okay** | Nudges are correct but timing is awkward. Openers need significant rewriting. |
| **Poor** | User feels surveilled or embarrassed by the nudges. Openers feel generic or transactional. |

## time-advisor

| Rating | Criteria |
|--------|----------|
| **Good** | Weekly reflections point to real patterns the user recognizes. Overcommitment flags catch problems before they bite. Deep work stays protected in aggregate. |
| **Okay** | Observations are accurate but the user doesn't act on them. Advice feels abstract. |
| **Poor** | Advice is generic ("try time-blocking"). User disengages from reviews. |

## prep-meeting

| Rating | Criteria |
|--------|----------|
| **Good** | Brief arrives before the meeting with exactly the context the user needed. Attendee notes are accurate. Past thread pull is on-point. |
| **Okay** | Brief is useful but includes irrelevant context. User still opens the email thread themselves. |
| **Poor** | Brief has fabricated information, or misses obvious context (like the shared doc the meeting is about). |

## Overall Role Success

The personal assistant role is succeeding if, over a month, the user can say:

1. **"Nothing important slipped."** — No missed deadlines, no double-bookings, no forgotten follow-ups.
2. **"My deep work survived."** — Peak hours spent on top-3 goals, not triage.
3. **"I'm not managing the assistant."** — The cognitive load of using it is lower than the load it removes.
4. **"Interrupts were rare and earned."** — When the agent spoke up mid-day, the user was glad.
5. **"The agent knows me."** — Axioms are referenced in recommendations; no generic advice.

## Failure Signals to Watch

- The user creates a shadow task list outside the system → the agent isn't trusted.
- The user silences morning briefs → the brief is too long or too noisy.
- The user re-checks every suggestion manually → suggestions are being inferred from missing context.
- The user stops giving feedback → the loop has become invisible; re-introduce explicit thumbs or short postmortems.
