# Learning Science — Foundations

The research-backed principles this tutor operates on. Skills reference this file by name; if you change the principles here, skills downstream will change behavior.

## Zone of Proximal Development (Vygotsky)

Learning happens at the edge of what the student can do with help but not yet alone. Too easy → boredom, no growth. Too hard → overwhelm, no encoding. The tutor's job is to keep tasks at **mastery + ε** — measurably above current competence, within reach with scaffolding.

**Operational rule:** target item difficulty ≈ current mastery. Drift up as mastery rises; drop back after errors.

## Cognitive Load Theory (Sweller)

Working memory holds ~4 chunks. Learning fails when total load exceeds capacity. Three load types:

- **Intrinsic** — inherent to the material. Can't be reduced without changing the content.
- **Extraneous** — imposed by *how* material is presented. Always reducible. Bad design = high extraneous load.
- **Germane** — load from actually processing for understanding. This is the load we *want*.

**Operational rule:** before adding a feature, ask "does this reduce extraneous load?" Before adding a concept, ask "does the student have the chunks to hold the prerequisites plus this?"

## Retrieval Practice (the Testing Effect)

The act of recalling strengthens the memory more than re-reading does. This effect is large and reliable — one of the most replicated findings in cognitive psychology (Roediger & Karpicke, 2006 onward).

**Implication:** the most efficient study activity is *trying to remember*, not re-reading. Quizzes aren't grading instruments here; they're learning instruments.

**Operational rule:** every teaching session ends with retrieval. Synthesis ends with a retrieval hook. New material is encoded, then retrieved within 10 minutes — not 10 days.

## Spacing Effect

Distributed practice beats massed practice for long-term retention. Cramming produces high immediate scores that decay fast. Spacing produces slightly lower immediate scores that decay slowly — and the long-term integral is much larger.

**Operational rule:** SRS schedules expand intervals. Don't let the student cram concepts back-to-back when the goal is durable learning.

**Exception:** for imminent-exam cramming (days, not weeks), massing + final-week interleaving is the evidence-supported approach.

## Interleaving

Mixing problem types within a practice session — as opposed to blocking one type — produces weaker in-session performance but stronger long-term retention and *transfer*. Interleaving trains the discrimination skill: which method, which concept, which tool fits this problem?

**Two-phase rule** (Birnbaum et al., consolidated in the 2021 Springer review):
- **During initial acquisition:** block. New concepts need focused attention to encode at all.
- **During review:** interleave. Discrimination matters more than encoding.

**Operational rule:** new concepts block; review mixes.

## Worked-Example Effect

For novices, studying a fully worked example then producing a variant outperforms solving a problem cold. The effect *reverses* for experts — the expertise-reversal effect — because experts are slowed by the worked structure.

**Operational rule:**
- mastery < 0.3 → worked examples
- 0.3 ≤ mastery < 0.7 → guided practice (I-do / we-do / you-do ladder)
- mastery ≥ 0.7 → independent / Socratic

## Desirable Difficulty (Bjork)

Learning conditions that feel *harder in the moment* often produce *stronger* learning. Retrieval practice, spacing, and interleaving all feel like worse study than re-reading; all produce better retention.

**Operational rule:** the tutor defends the discomfort when it's the right kind of difficulty, and names what the student is experiencing so they don't misread it as failure.

## Dual Coding

Memory is stronger when encoded through two channels (verbal + visual, for example). Diagrams paired with prose outperform either alone for most learners.

**Operational rule:** when a concept has a natural visual, include one. When synthesis is dense, add a concept map.

## Elaboration

Connecting new material to prior knowledge ("how is this like / unlike X I already know?") deepens encoding. Rote memorization without elaboration produces isolated, brittle memories.

**Operational rule:** when teaching, always ask "where does this connect to what you already know?" Use cognitive analogies — they *are* elaborations.

## Metacognition

Students routinely misjudge what they know. Feeling familiar ≠ being able to reproduce. Calibration — the match between felt confidence and actual competence — is a learnable skill and a strong predictor of independent learning success.

**Operational rule:** the tutor surfaces calibration explicitly. "How confident are you?" paired with "let's check" builds the skill over time.

## Microlearning

Retention per minute is higher in short focused blocks (5-15 min) than in long sessions. There's a cliff past ~25 minutes where returns collapse.

**Operational rule:** default chunk 10 minutes, hard cap 25. Break enforced after. Longer sessions only when explicitly requested and the student is in a high-engagement window.

## Transfer-Appropriate Processing

Memory retrieval is best when retrieval conditions match encoding conditions. Students taught only via multiple-choice struggle with short-answer of the same material.

**Operational rule:** vary item formats so encoding matches the breadth of situations the student will face in the real world (or on the exam).
