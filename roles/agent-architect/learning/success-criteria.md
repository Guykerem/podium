# Agent Architect — Success Criteria

How to evaluate whether each skill is performing well.

## guide-agent-design

| Rating | Criteria |
|--------|----------|
| **Good** | User completes all 6 questions with specific, actionable answers. Design summary is clear enough to build from. User identifies a realistic critical element. |
| **Okay** | User completes the flow but answers are vague or generic. Design summary needs refinement before building. |
| **Poor** | User drops out mid-flow, feels overwhelmed, or produces a design that wouldn't work as an agent (fails the agent test). |

## explain-architecture

| Rating | Criteria |
|--------|----------|
| **Good** | User can explain the concept in their own words after the explanation. They open the referenced file and understand what they're looking at. |
| **Okay** | User understands the explanation but can't rephrase it independently. They see the file but need help connecting it to the concept. |
| **Poor** | User is confused after the explanation, or the analogy didn't land. They don't see why the file matters. |

## recommend-role

| Rating | Criteria |
|--------|----------|
| **Good** | User feels confident in their role choice. The recommendation clearly matches their stated needs. They move forward to customization or onboarding. |
| **Okay** | User picks a role but seems uncertain. The match is reasonable but not precise. |
| **Poor** | User feels pushed toward a role that doesn't fit, or can't decide despite the guidance. The recommendation missed their actual need. |

## customize-role

| Rating | Criteria |
|--------|----------|
| **Good** | User completes at least 4 of 6 customization steps. They make deliberate choices (not just accepting defaults). The result reflects their needs. |
| **Okay** | User completes the flow but mostly accepts defaults. Customization is minimal — the role doesn't feel personalized. |
| **Poor** | User gets lost in options, feels overwhelmed by the checklist, or makes changes they don't understand. |

## teach-concepts

| Rating | Criteria |
|--------|----------|
| **Good** | User grasps the concept, connects it to the cognitive analogy, and can point to the relevant file. They ask a follow-up question that shows deeper engagement. |
| **Okay** | User understands the concept at surface level but doesn't connect it to other concepts or explore the files. |
| **Poor** | User is confused, the analogy didn't help, or they feel talked at rather than taught. |

## Overall Success

The agent architect role is succeeding if users can:

1. **Explain** what an agent is and how it differs from a chatbot
2. **Design** an agent for a specific use case using the 6-question framework
3. **Configure and run** an agent by editing files in this repo
4. **Evaluate** whether a use case is right for an agent (the agent test)
