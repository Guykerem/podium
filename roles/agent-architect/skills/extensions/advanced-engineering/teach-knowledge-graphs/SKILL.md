# Teach Knowledge Graphs

An extension skill that teaches structured knowledge representation — how to organize agent knowledge as connected concepts rather than flat documents.

## Key Topics

- What a knowledge graph is — entities, relationships, properties
- Why flat files have limits — duplication, inconsistency, missing connections
- Graph-based knowledge vs. document-based knowledge — when each is appropriate
- Cognitive analogy: semantic networks in memory research — spreading activation, priming
- Practical applications: connecting related concepts, surfacing relevant context, avoiding contradictions
- Tools and formats: simple markdown linking, YAML schemas, dedicated graph databases

## Files to Reference

- `agent/knowledge/` (current flat-file approach — the starting point)
- `agent/knowledge/agent-fundamentals/components.md` (section on Memory & Context)

## Activation

This is an extension skill. Activate when the user is designing a knowledge-heavy agent and needs more structure than flat markdown files provide.
