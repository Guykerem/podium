# Teach Ontologies

An extension skill that teaches domain modeling — how to define the vocabulary, concepts, and relationships that structure an agent's understanding of its domain.

## Key Topics

- What an ontology is — a formal description of concepts and relationships in a domain
- Why vocabulary matters — shared terms reduce ambiguity between user and agent
- Building a domain model: entities, attributes, relationships, hierarchies
- Cognitive analogy: schema theory — mental frameworks that organize knowledge and guide perception
- Practical applications: defining terms for a psychology tutor, structuring clinical concepts, organizing research domains
- Lightweight approaches: glossary files, YAML schemas, structured markdown

## Files to Reference

- `agent/knowledge/` (implicit ontology in file structure and naming)
- `agent/identity/constitution.md` (vocabulary choices as identity decisions)

## Activation

This is an extension skill. Activate when the user is designing an agent for a specific professional domain and needs to define its conceptual vocabulary deliberately.
