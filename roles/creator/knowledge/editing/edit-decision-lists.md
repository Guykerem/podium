# Edit Decision Lists

EDL / FCPXML / OTIO — formats for handing cuts off between the agent and desktop NLEs (Premiere, Final Cut Pro, DaVinci Resolve).

## When to emit an EDL vs rendering directly

- **Render directly (FFmpeg / MoviePy / Remotion)** — for shorts, templated content, programmatic branded video, anything the creator won't touch after
- **Emit an EDL** — when the creator wants to finish the edit in Resolve / Premiere / FCP, adding polish (J-cuts, audio sweetening, color grading) the agent shouldn't auto-apply

## OpenTimelineIO (OTIO) — the canonical format

Pixar's open format. Converts between EDL, FCPXML, AAF, Premiere XML. **Always keep OTIO as the canonical format; emit target-specific formats from it.**

```python
import opentimelineio as otio

tl = otio.schema.Timeline(name="My Edit")
v_track = otio.schema.Track(kind=otio.schema.TrackKind.Video)
a_track = otio.schema.Track(kind=otio.schema.TrackKind.Audio)
tl.tracks.extend([v_track, a_track])

for seg in segments:
    clip = otio.schema.Clip(
        name=seg["id"],
        media_reference=otio.schema.ExternalReference(target_url=f"file://{seg['source']}"),
        source_range=otio.opentime.TimeRange(
            start_time=otio.opentime.RationalTime(seg["in"] * 30, 30),
            duration=otio.opentime.RationalTime((seg["out"] - seg["in"]) * 30, 30),
        ),
    )
    v_track.append(clip)

# Convert to target formats
otio.adapters.write_to_file(tl, "edit.otio")
otio.adapters.write_to_file(tl, "edit.fcpxml")
otio.adapters.write_to_file(tl, "edit.edl")
```

## FCPXML — Final Cut Pro + DaVinci Resolve

Modern, rich, well-supported in Resolve (and Premiere via import).

```xml
<?xml version="1.0" encoding="UTF-8"?>
<fcpxml version="1.11">
  <resources>
    <format id="r1" frameDuration="100/3000s" width="1920" height="1080"/>
    <asset id="r2" src="file:///path/to/clip.mp4" duration="600s" hasVideo="1" format="r1" hasAudio="1"/>
  </resources>
  <library>
    <event name="Project">
      <project name="Main">
        <sequence format="r1">
          <spine>
            <asset-clip ref="r2" offset="0s" duration="5s" start="0s"/>
            <asset-clip ref="r2" offset="5s" duration="8s" start="12s"/>
          </spine>
        </sequence>
      </project>
    </event>
  </library>
</fcpxml>
```

Import: FCP → File → Import → XML. DaVinci: File → Import Timeline → Import AAF/EDL/XML.

## EDL — CMX3600

Oldest format. Line-based. Limited (no effects, no keyframes). Use only when target tool doesn't support FCPXML.

```
TITLE: My Edit
FCM: NON-DROP FRAME

001  AX       V     C        00:00:00:00 00:00:05:00 01:00:00:00 01:00:05:00
001  AX       A     C        00:00:00:00 00:00:05:00 01:00:00:00 01:00:05:00
* FROM CLIP NAME: clip1.mp4

002  AX       V     C        00:00:12:00 00:00:20:00 01:00:05:00 01:00:13:00
002  AX       A     C        00:00:12:00 00:00:20:00 01:00:05:00 01:00:13:00
* FROM CLIP NAME: clip1.mp4
```

Format: `event# reel track transition src_in src_out dest_in dest_out`

## Premiere XML

Adobe's format. OTIO can emit it. Also generated by Premiere's own export.

## AAF — Avid Advanced Authoring Format

Enterprise broadcast. Rarely needed for creator work unless handing to a professional editor.

## Tool support matrix

| Tool | FCPXML | EDL | OTIO | Premiere XML | AAF |
|---|---|---|---|---|---|
| Final Cut Pro | native | via convert | via plugin | no | no |
| DaVinci Resolve | import/export | import/export | via Python | import/export | import/export |
| Premiere Pro | import/export (older ver) | import/export | via Python | native | import/export |
| Avid Media Composer | via convert | import | via Python | via convert | native |
| Remotion | no (programmatic alternative) | no | no | no | no |

## Agent workflow

1. `edit-video` generates `segments.json` or `edit.yaml` cut list
2. Build OTIO timeline in memory
3. Emit both an auto-rendered MP4 (creator can use as-is) AND an OTIO + FCPXML (creator can finish in NLE)
4. Package:
   ```
   edits/<slug>/
     auto_render.mp4          # agent's auto-cut
     edit.otio                # canonical
     edit.fcpxml              # FCP / Resolve
     edit.edl                 # fallback
     edit.xml                 # Premiere
     README.md                # how to import
   ```

## README template for handoff

```markdown
# Edit Handoff — {slug}

## Auto-render
`auto_render.mp4` is the agent's edit. You can publish as-is if it's good.

## Finishing in your NLE

**DaVinci Resolve**:
1. File → Import Timeline → Import AAF/EDL/XML
2. Select `edit.fcpxml`
3. Point to source media if prompted

**Final Cut Pro**:
1. File → Import → XML
2. Select `edit.fcpxml`

**Premiere Pro**:
1. File → Import → select `edit.xml`
2. Relink media when prompted

## Source media
{list of source files}
```
