# Brainstorm Agent Skill

## What This Skill Does

A guided flow that walks the student through designing their own AI agent. This is the workshop skill — it structures the creative process of going from "I have an idea" to "I have a design I can build."

## When To Use

- During the workshop section of the lecture
- Any time a student says "I want to build an agent that..."
- When a student has a vague idea and needs help making it concrete

## How It Works

Walk through these six questions, one at a time. Don't rush. Each question is a thinking prompt — give the student time to work through it.

### Step 1: What's your use case?
"Describe a recurring challenge in your life — as a student, a future practitioner, or just a person. Something you do regularly that could benefit from an AI partner."

Check it against the good use case criteria:
- Recurring?
- Memory-dependent?
- Clear success criteria?
- Structured steps?
- Recoverable mistakes?

If it fails multiple criteria, gently redirect: "That's an interesting idea. Let me push back on one thing..."

### Step 2: What does it need to KNOW?
"What information does this agent need access to? Think about: domain knowledge, personal context about you, external sources."

Help them distinguish between:
- Static knowledge (reference material, domain expertise)
- Dynamic context (what happened in past interactions, user preferences)
- External data (APIs, databases, real-time information)

### Step 3: What does it need to DO?
"Walk me through the loop. What does the agent observe, think about, decide, and do?"

Help them sketch the agentic loop:
- Observe: what triggers the agent? What inputs?
- Think: what reasoning or analysis?
- Decide: what choices or judgments?
- Act: what outputs or actions?
- Learn: what happens after the action?

### Step 4: What should it FEEL LIKE?
"If this agent were a person, what kind of person? What's the vibe?"

Explore:
- Communication style (warm ↔ formal, concise ↔ detailed)
- Personality traits (challenger, supporter, teacher, coach)
- Non-negotiable behaviors (things it should always or never do)

### Step 5: How does it LEARN?
"How does the agent know if it did a good job? How does it improve?"

Define:
- Success criteria — what does "good output" look like?
- Feedback mechanism — how does the student signal quality?
- Adaptation — what should change based on feedback?

### Step 6: What's the CRITICAL ELEMENT?
"Which one component, if you got it wrong, would break the whole agent? Why? And how hard is it to get right?"

This is the most important question. Push them to be specific:
- Why is this the maker-or-breaker?
- What would they need to get it right?
- Do they have access to that? Is it feasible?

## After the Flow

Produce a summary of the agent design using the workshop template format. Offer to help them start building — point them to the relevant skill directory as a template.

## Autonomy Behavior

- **Level 1:** Ask each question, wait for the answer, then move to the next. Full guided flow.
- **Level 2:** Same as level 1 — this skill is inherently interactive.
- **Level 3:** Same as level 1 — this skill is inherently interactive.

This skill always runs at effective level 1 regardless of the autonomy setting. The thinking is the point.

## Cognitive Analogy

This skill mirrors **metacognitive planning and self-regulation** — the ability to think about your own thinking, identify what you need, and structure a plan before acting. It's the prefrontal cortex at its best: not executing, but designing the execution.
